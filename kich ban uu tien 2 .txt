## PHASE 1 — EMAIL PHISHING (Ngày 1, 09:00)

### Muc tieu
Gui email phishing tu domain gia `sakura-vendor.com` (typosquat cua `sakura-vendor.net`) den chi Linh, kem link dan den trang gia mao.

### Ky thuat su dung
- MITRE T1566.002 — Spearphishing Link
- Domain typosquatting — sakura-vendor**.com** thay vi **.net**
- Email spoofing — From header gia mao
- Social engineering — Urgent language, impersonation CEO

### Setup GoPhish tren 101

```bash
# SSH vao 101
cd /tmp
wget https://github.com/gophish/gophish/releases/download/v0.12.1/gophish-v0.12.1-linux-64bit.zip
apt install -y unzip
unzip gophish-v0.12.1-linux-64bit.zip -d /opt/gophish
chmod +x /opt/gophish/gophish
cd /opt/gophish

# Chinh config.json — cho phep truy cap tu ngoai
sed -i 's/127.0.0.1:3333/0.0.0.0:3333/' config.json

# Chay GoPhish
./gophish &
```

Truy cap GoPhish admin: `https://<IP_101>:3333`
- Default login: `admin / gophish`

## PHASE 2 — USER INTERACTION (Ngày 1, 09:15)

### Muc tieu
Chi Linh click link trong email → trang canh bao hien ra → GoPhish log lai hanh dong.

### Ky thuat su dung
- MITRE T1566.002 — User clicks phishing link
- Tracking pixel — Ghi lai IP, timestamp khi email duoc mo
- GoPhish campaign tracking — Log click event

### Simulate tu may victim (100 srv-ubuntu)

```bash
# Cai browser headless de simulate victim click
apt install -y curl

# Simulate mo email (tracking pixel fired)
curl -s "http://<IP_101>/track/open?rid=<campaign_id>" > /dev/null

# Simulate click link
curl -s "http://<IP_101>/track/click?rid=<campaign_id>" > /dev/null
```

### Dau vet sinh ra (IOC Phase 2)

| Log Source | Noi dung | Forensic Value |
|---|---|---|
| GoPhish log | Email opened — IP victim, timestamp | Xac nhan nan nhan mo email |
| GoPhish log | Link clicked — IP victim, user-agent | Xac nhan nan nhan click |
| DNS log (pfSense) | Query den domain las login-sakura-vendor.com | Truy vet domain phishing |
| Nginx access log (101) | GET request tu IP victim | Timeline click |
| Wazuh alert | DNS query to suspicious domain | Alert tu dong |

## PHASE 3 — ACCOUNT COMPROMISE (Ngày 1, 10:30)

### Muc tieu
Simulate attacker dang nhap mailbox chi Linh tu IP nuoc ngoai.

### Ky thuat su dung
- MITRE T1078 — Valid Accounts
- Impossible Travel — Login VN luc 09:15, login "Dong Au" luc 10:30

### Script simulate tren 100 srv-ubuntu

```bash
# Simulate login binh thuong tu IP noi bo (victim)
curl -k -X POST https://10.71.119.51/mail \
  -d "user=linhntt@sakuratech.local&pass=Victim@2026!" \
  --header "X-Forwarded-For: 10.71.119.100" \
  --header "User-Agent: Mozilla/5.0 (Windows NT 10.0)"

# Simulate login attacker tu IP nuoc ngoai (inject vao log)
logger -p auth.info -t dovecot \
  "imap-login: Login: user=<linhntt@sakuratech.local>, \
  method=PLAIN, rip=185.220.101.47, lip=10.71.119.51, \
  mpid=1234, secured, session=<attacker_session>"
```

### Inject impossible travel log vao Wazuh

```bash
# Tao file log gia lap tren 107
cat >> /var/log/dovecot.log << 'EOF'
2024-01-15T10:30:02 imap-login: Login: user=<linhntt@sakuratech.local>, method=PLAIN, rip=185.220.101.47, lip=10.71.119.51, mpid=5678, secured
EOF
```

### Dau vet sinh ra (IOC Phase 3)

| Log Source | Noi dung | Forensic Value |
|---|---|---|
| Dovecot auth log | Login tu 185.220.101.47 | IP attacker |
| Dovecot auth log | 2 sessions dong thoi | Impossible travel |
| Wazuh alert | Rule 18152 — multiple sessions | Alert tu dong |
| GeoIP lookup | 185.220.101.47 → Dong Au (Tor exit) | Vi tri attacker |

### Ky thuat su dung

## QUY TRINH DIEU TRA TU KALI (113 jump-kali)

### Buoc 1 — Thu thap chung cu

```bash
# SSH vao mail server
ssh root@10.71.119.51

# Export email goc (.eml)
find /var/vmail/sakuratech.local/linhntt/ -name "*.eml" -newer /tmp/incident_time

# Hash chung cu (chain of custody)
md5sum suspicious_email.eml > suspicious_email.eml.md5
sha256sum suspicious_email.eml > suspicious_email.eml.sha256

# Collect logs
cp /var/log/mail.log /tmp/evidence/
cp /var/log/dovecot.log /tmp/evidence/
cp /var/log/nginx/access.log /tmp/evidence/
```

### Buoc 2 — Phan tich Email Header

```bash
# Cai cong cu phan tich
apt install -y emailheader

# Parse header tu .eml file
cat suspicious_email.eml | grep -A 50 "Received:"

# Truy vet chuoi Received (doc tu duoi len)
# Received: from mail.sakura-vendor.com (185.220.101.47)
# → IP nguon that cua attacker
```

**Tool online de demo trong bao cao:**
- Google Admin Toolbox: `toolbox.googleapps.com/apps/messageheader`
- MXToolbox: `mxtoolbox.com/EmailHeaders.aspx`

### Buoc 3 — Truy vet IP Attacker

```bash
# WHOIS lookup
whois 185.220.101.47

# GeoIP lookup
curl https://ipinfo.io/185.220.101.47/json

# Check reputation
# Tra cuu tren VirusTotal, Cisco Talos, AbuseIPDB
```

**Ket qua dieu tra:**
```
IP: 185.220.101.47
ASN: AS24940 (Hetzner Online)
Country: Germany (DE)
Type: Tor exit node
Reputation: MALICIOUS (AbuseIPDB: 95% confidence)
```

### Buoc 4 — Kiem tra SPF/DKIM/DMARC

```bash
# Kiem tra SPF record cua domain gia
dig TXT sakura-vendor.com

# Kiem tra DKIM
dig TXT default._domainkey.sakura-vendor.com

# Kiem tra DMARC
dig TXT _dmarc.sakura-vendor.com
```

**Ket qua:**
```
SPF:   FAIL — domain not authorized to send
DKIM:  FAIL — no signature found
DMARC: NONE — policy not configured
```

## WAZUH CUSTOM RULES

Them vao `/var/ossec/etc/rules/local_rules.xml` tren Wazuh server:

```xml
<!-- Rule: Impossible Travel Detection -->
<rule id="100001" level="12">
  <if_sid>5501</if_sid>
  <match>Login</match>
  <regex>rip=185\.220\.101\.</regex>
  <description>Suspicious login from known Tor exit node</description>
  <group>authentication_failed,pci_dss_10.2.4,</group>
</rule>

<!-- Rule: Suspicious Mail Forward Rule -->
<rule id="100002" level="10">
  <if_sid>5501</if_sid>
  <match>sieve</match>
  <match>fileinto|redirect</match>
  <description>New mail forwarding rule created</description>
  <group>mail,suspicious,</group>
</rule>

<!-- Rule: BEC Detection — External IP sending internal mail -->
<rule id="100003" level="14">
  <if_sid>5501</if_sid>
  <match>postfix/smtp</match>
  <regex>from=.*sakuratech\.local.*relay=.*185\.220</regex>
  <description>Internal email sent from external IP — possible BEC</description>
  <group>mail,bec,high_severity,</group>
</rule>
```
